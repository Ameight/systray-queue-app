# Queue — task queue in the menu bar

Приложение-очередь задач в системном трее для macOS. Задачи хранятся локально, поддерживают Markdown и вложения (изображения, аудио). Управление — через меню трея, горячие клавиши и браузерный интерфейс.

---

## Установка

### Готовый релиз (рекомендуется)

1. Скачать последний релиз со страницы [Releases](https://github.com/Ameight/systray-queue-app/releases)
2. Открыть скачанный `.dmg`, перетащить **Queue.app** в `~/Applications`
3. Запустить `Queue.app`

> **Gatekeeper.** При первом запуске macOS заблокирует неподписанное приложение.
> Откройте **System Settings → Privacy & Security → Open Anyway**, или выполните в терминале:
> ```bash
> xattr -dr com.apple.quarantine ~/Applications/Queue.app
> ```

### Сборка из исходников

**Требования:**
- Go 1.21+
- macOS (Apple Silicon или Intel)
- Xcode Command Line Tools: `xcode-select --install`

```bash
git clone https://github.com/Ameight/systray-queue-app
cd systray-queue-app

# Собрать бинарь (arm64)
make build

# Собрать .app бандл (рекомендуется)
make bundle

# Открыть
open Queue.app
```

Для разработки без сборки:
```bash
make run
```

## Обновления

Приложение автоматически проверяет наличие обновлений при запуске и раз в сутки в фоне. При обнаружении новой версии появляется системное уведомление.

Установить обновление можно из **Settings → Updates**:
- Кнопка **Check for updates** — ручная проверка
- Кнопка **Install and restart** — скачивает новый бинарь, заменяет текущий и перезапускает приложение

Обновление не требует прав администратора, если приложение установлено в `~/Applications`.

## Меню трея

| Пункт | Действие |
|---|---|
| `<название задачи>` | Открыть текущую задачу в браузере |
| **Start timer** | Запустить / паузить Pomodoro-таймер |
| **Skip** | Переместить текущую задачу в конец очереди |
| **Done** | Завершить текущую задачу и добавить в историю |
| **Add task…** | Быстрое добавление через диалог |
| **Add task (advanced)…** | Расширенный редактор в браузере |
| **View current task…** | Просмотр текущей задачи в браузере |
| **Manage order…** | Список всех задач, сортировка, редактирование |
| **Settings…** | Горячие клавиши, трей, автозапуск, обновления |
| **Quit** | Выйти из приложения |

Видимость и порядок групп меню настраиваются в **Settings → Трей**.

## Добавление задачи

**Быстрое добавление** (меню → *Add task…*): системный диалог с текстом. Поддерживает Markdown.

**Расширенный редактор** (меню → *Add task (advanced)…*): открывается в браузере.

- Поддержка Markdown с предпросмотром
- Прикрепить файл: кнопка выбора файла (изображения и аудио)
- **Вставить изображение из буфера**: нажать `⌘V` / `Ctrl+V` в поле текста — изображение автоматически прикрепляется как вложение
- **Записать голосовую заметку**: кнопка *Record voice note* — запись через микрофон, сохраняется как аудио-вложение

Поддерживаемые форматы вложений: `.png`, `.jpg`, `.jpeg`, `.webp`, `.gif`, `.mp3`, `.m4a`, `.wav`, `.ogg`.

## Управление очередью (Manage order)

Страница открывается через меню → *Manage order…* или горячей клавишей.

- **Перетаскивание** элементов — изменить порядок, **Save order** — сохранить
- **Клик по задаче** — открыть предпросмотр в правой панели
- В предпросмотре:
  - **Edit** — редактировать текст; при редактировании `⌘V` / `Ctrl+V` вставляет изображение из буфера как вложение
  - **Done** — завершить задачу и отправить в историю
  - **Delete** — удалить задачу без сохранения в историю

## Голосовые заметки

Кнопка *Record voice note* доступна всегда. Поведение зависит от настройки **Whisper**:

| Whisper | Результат |
|---|---|
| Выключен | Запись сохраняется как аудио-вложение без транскрипции |
| Включён | Запись транскрибируется, текст добавляется в поле задачи; аудио сохраняется как вложение |

Для транскрипции нужен [openai-whisper](https://github.com/openai/whisper):
```bash
pip install openai-whisper
```

Включить транскрипцию: **Settings → Voice transcription (Whisper)**.

## Taймер

Pomodoro-таймер в меню трея. Длительность настраивается в **Settings → Трей → Длительность таймера** (по умолчанию 25 минут). По истечении — системное уведомление.

Таймер работает независимо от очереди задач. В заголовке меню бара отображается:
- Обратный отсчёт таймера (если запущен)
- Время с начала текущей задачи (если прошло больше минуты)

## Горячие клавиши

Настраиваются в **Settings** или напрямую в файле конфига.

| Комбинация по умолчанию | Действие |
|---|---|
| `Ctrl+Alt+Q` | Открыть текущую задачу |
| `Ctrl+Alt+A` | Расширенный редактор добавления |
| `Ctrl+Alt+N` | Быстрое добавление |
| `Ctrl+Alt+S` | Пропустить текущую задачу |
| `Ctrl+Alt+D` | Завершить текущую задачу |
| `Ctrl+Alt+M` | Открыть управление очередью |

Конфиг создаётся при первом запуске: `~/Library/Application Support/systray-queue-app/key-config.yaml`.

Поддерживаемые модификаторы: `ctrl`, `alt` / `option`, `shift`, `cmd`.
Поддерживаемые клавиши: `a`–`z`, `0`–`9`, `f1`–`f12`.

## Автозапуск

**Settings → Autostart** — добавить в автозагрузку при входе в систему.

Работает через LaunchAgent: `~/Library/LaunchAgents/com.ameight.queue.plist`.

## Данные приложения

Все данные хранятся в `~/Library/Application Support/systray-queue-app/`:

```
systray-queue-app/
├── queue.json          # активная очередь
├── history.json        # завершённые задачи
├── attachments/        # вложения (изображения, аудио)
└── key-config.yaml     # настройки горячих клавиш и трея
```

Файлы `queue.json` и `history.json` — обычный JSON, можно редактировать вручную.

## Сборка и выпуск релиза

```bash
# Собрать бинарь для arm64
make build VERSION=v1.2.0

# Собрать .app бандл
make bundle VERSION=v1.2.0

# Создать DMG для дистрибуции
make dmg VERSION=v1.2.0
# → Queue-v1.2.0-arm64.dmg

# Собрать бинарники для обеих архитектур (нужны для авто-обновления)
make release-assets VERSION=v1.2.0
# → systray-queue-app-darwin-arm64
# → systray-queue-app-darwin-amd64

# Universal binary (arm64 + amd64)
make build-universal VERSION=v1.2.0
```

### Публикация релиза на GitHub

1. `git tag v1.2.0 && git push --tags`
2. `make release-assets VERSION=v1.2.0`
3. Создать релиз на GitHub с тегом `v1.2.0` и прикрепить:
   - `systray-queue-app-darwin-arm64`
   - `systray-queue-app-darwin-amd64`
   - `Queue-v1.2.0-arm64.dmg` (опционально)

Пользователи с уже установленным приложением получат уведомление и смогут обновиться одной кнопкой из Settings.

> **Версия по умолчанию** берётся из `git describe --tags`. Если тегов нет — `dev`.
